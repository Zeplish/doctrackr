Build a production-ready MVP web application from scratch called:

DocTrackr

Tagline:
Never miss an important document renewal.

Purpose:
DocTrackr is a HIPAA-conscious document expiry tracking and reminder system for daycare/school operations. It does NOT store completed student/employee documents. It only tracks document names, required document checklists, expiry dates, reminder status, and sends branded reminder emails.

Important Product Rule:
Do NOT build a system for storing completed documents.
Do NOT upload, view, download, or replace completed student/employee documents.
The daycare keeps physical copies separately.
The software only tracks expiry/compliance status.

However:
Admin should be able to upload blank template forms for student document types only, so those blank forms can be attached to parent reminder emails.

CORE PRODUCT CONCEPT
==================================================

Each person has required document checklist items.

Students have fewer required documents.
Employees have many required documents.

The admin should not manually add every document one by one.

Instead:

Add Student
→ View Student Profile
→ System automatically shows required student document checklist
→ Admin enters expiry dates in a table

Add Employee
→ View Employee Profile
→ System automatically shows required employee document checklist
→ Admin enters expiry dates in a table

Goal:
Admin should be able to update 32 employee document expiry dates in under 2 minutes.

==================================================
REMINDER LOGIC
==================================================

Replace old 7-day reminder logic.

New reminder schedule:

Start sending reminders 30 days before expiry.

Send every 3 days until expiry date.

Example:
Expiry Date = May 31

Send reminders on:
May 1
May 4
May 7
May 10
May 13
May 16
May 19
May 22
May 25
May 28
May 31

After expiry:
Continue reminders every 7 days until expiry date is updated.

Cron:
- Run daily
- Recommended default time: 9 AM server time
- Configurable in settings if practical

Duplicate protection:
- Do not send more than one reminder per checklist item per day
- Use email_logs table
- Track sent reminders carefully
- Prevent duplicate sends if cron runs twice

Reminder eligibility:
- Only active students/employees
- Only checklist items with expiry date
- Only documents that are within reminder window or overdue

When expiry date is updated:
- Recalculate status
- Recalculate next reminder due date
- Continue future reminders based on new expiry date

Manual reminder:
Admin should have “Send Reminder Now” action for checklist items.
